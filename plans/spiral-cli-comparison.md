# Spiral CLI Comparison Plan

Purpose: add an explicit side-by-side comparison path for MacKenzie Voice Tool outputs with and without Spiral CLI, using `@every-env/spiral-cli`.

## Context

The MacKenzie Voice Tool currently produces source-backed drafts and reviews using the local MacKenzie corpus, voice guide, grading schema, and feedback ledger. Spiral should be added as a comparison lane, not as a replacement for the existing tool.

The goal is to help Patch compare:

- baseline MacKenzie Voice Tool output
- Spiral-enhanced output from the same prompt/context

This should make it easier to judge whether Spiral improves voice fidelity, human feel, specificity, and usefulness.

## Spiral CLI Surface

Package:

```bash
@every-env/spiral-cli
```

Initial invocation:

```bash
bunx --bun @every-env/spiral-cli --help
```

Verified local note, 2026-05-16: `bunx --bun @every-env/spiral-cli --help`
works, but `bunx --bun @every-env/spiral-cli auth status` failed locally in a
transitive dependency path before reaching Spiral auth. Plain `bunx
@every-env/spiral-cli auth status` worked and returned `Not authenticated`.
The comparison wrapper should therefore support a configurable runner and fall
back from `bunx --bun` to plain `bunx` when command startup fails.

Relevant commands:

```bash
spiral write "prompt" --instant --json
spiral personalize "text to personalize" --json
spiral humanize "ai-sounding text" --json
spiral styles
spiral workspaces
spiral sessions
spiral quota
spiral auth status
spiral setup --pairing-code <code>
spiral auth login --token <key>
```

Do not store Spiral tokens in project files or browser state. Treat auth as local user setup.

## Recommended Integration Shape

Add Spiral as an optional local runner inside this project. Keep the first version simple and CLI-driven before wiring it into a larger UI.

Proposed local script:

```text
mackenzie-price-style/tools/compare_spiral_outputs.py
```

Responsibilities:

- load or accept a drafting prompt
- run the baseline MacKenzie Voice Tool prompt path
- run one or more Spiral commands against the same input
- write a markdown comparison report
- preserve raw Spiral JSON/stdout for inspection
- never send or publish output externally

For v1, do not automate the baseline drafting path. The MacKenzie Voice Tool is
currently a skill/context workflow, not a deterministic local generator. Accept
an explicit `--baseline-file` and compare Spiral against that known draft.

Possible output location:

```text
mackenzie-price-style/training/review-sessions/YYYY-MM-DD-spiral-comparison.md
```

## First Implementation Slice

1. Add a small CLI wrapper script.
   - Inputs: prompt text, optional input file, mode, output path.
   - Modes: `write`, `personalize`, `humanize`.
   - Default to `bunx @every-env/spiral-cli` so the project does not need a permanent npm dependency for v1.
   - Allow `--runner` override for local installs, `bunx --bun`, or a pinned wrapper.

2. Add a baseline lane.
   - For v1, baseline can be a saved draft generated by the existing MacKenzie Voice Tool workflow.
   - The script should accept `--baseline-file` so comparison can start without automating the entire drafting agent.

3. Add a Spiral lane.
   - `write`: use the original prompt with `--instant --json`.
   - `personalize`: use the baseline draft as input.
   - `humanize`: use the baseline draft as input.
   - Optional context: attach `mackenzie-voice-context-pack.md` with `--file` for `write` tests, but keep `personalize` and `humanize` focused on the baseline draft unless results show they need more context.

4. Produce a comparison report.
   - Include prompt, timestamp, command, elapsed time, exit code, stderr if present, parsed output, and raw output path.
   - Include auth/setup failures as first-class report results instead of crashing.
   - Include a scoring checklist aligned to the local grading schema:
     - sounds_like_mackenzie
     - clarity
     - conviction
     - parent_resonance
     - alpha_accuracy
     - not_generic
     - too_salesy
     - too_corporate
     - forbidden_phrases
     - keeper_phrases

5. Add a README note.
   - Document Spiral setup.
   - Document the comparison command.
   - Keep token/auth instructions local and non-secret.

## Later UI Option

After the CLI comparison is useful, add it to the GBrain/OpenClaw cockpit or another local UI as a side-by-side panel. The UI should call a server-side adapter rather than shelling out from the browser.

That later adapter should expose:

```text
POST /admin/api/spiral/run
GET /admin/api/spiral/status
```

But the MacKenzie Voice Tool should own the comparison prompt/report format first.

## Auth And Safety

- Require local Spiral authentication before running comparisons.
- Do not collect Spiral API keys in prompts, markdown files, browser local storage, or committed config.
- If Spiral is unauthenticated, fail with a setup hint.
- Keep outputs framed as drafts for human review.
- Do not auto-send any generated copy.
- Preserve the MacKenzie voice boundary: MacKenzie Price only, not Rachel or generic Alpha brand voice.

- run `bunx @every-env/spiral-cli auth status`
- run one tiny `humanize` or `personalize` smoke test
- verify timeout/nonzero-exit behavior
- generate one report from a known baseline draft
- manually compare against the local grading schema

- `bunx @every-env/spiral-cli --version`: pass, version 1.5.0

- `humanize`, `personalize`, and `write --instant` with `--json`: reachable but blocked by authentication, returning JSON `AuthenticationError`

## Open Questions

- Which Spiral command is most useful for the MacKenzie workflow: `write`, `personalize`, or `humanize`?
- Should Spiral receive the full context pack, the baseline draft only, or a compact prompt plus baseline draft?
- Should comparison reports be committed as training artifacts, or treated as ephemeral review outputs?
- Should future accepted Spiral-improved drafts be captured in the feedback ledger?
