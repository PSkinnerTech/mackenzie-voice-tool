# MacKenzie Voice Tool

Version: **v0.3**

A source-backed drafting system for Alpha School communications in MacKenzie Price's voice.

The recommended workflow is to create a trained Spiral style from the included PDF, then use Spiral CLI to draft, personalize, or humanize text generated by Claude, Perplexity, ChatGPT, Cursor, Gemini, or another AI tool.

This repository includes:

- a MacKenzie-only voice corpus
- a Spiral training PDF
- setup instructions for Spiral, Claude, and Perplexity
- an installable Claude/OpenClaw-style skill
- example drafts and comparison artifacts
- local scripts for context packaging, feedback capture, and Spiral output review

The goal is high-quality draft generation for human review. This tool is not permission to impersonate MacKenzie or send unsupervised messages.

## Recommended Workflow

Use Spiral as the final writing-style layer.

1. Use Claude, Perplexity, ChatGPT, Cursor, Gemini, or another AI tool to create a factual baseline draft.
2. Run that draft through Spiral using the trained MacKenzie style.
3. Review the output against the MacKenzie Voice Tool rules before sending.

Spiral produced the strongest output in side-by-side testing because it can apply a dedicated trained style consistently from the CLI.

## Spiral Setup

The Spiral training PDF is included here:

~~~text
docs/mackenzie-voice-spiral-training-pack.pdf
~~~

Use that PDF to create the Spiral MacKenzie style.

### 1. Create a Spiral account

1. Open Spiral in your browser.
2. Create an account or sign in.
3. Create or choose the workspace where you want the MacKenzie style to live.

### 2. Install or run the Spiral CLI

The easiest way to run the CLI is with bunx:

~~~bash
bunx @every-env/spiral-cli --help
~~~

If bun is unavailable, use pnpm:

~~~bash
pnpm dlx @every-env/spiral-cli --help
~~~

You should see commands including write, personalize, humanize, styles, workspaces, and auth.

### 3. Authenticate the CLI

In Spiral, open the agent or CLI connection flow and copy the pairing code. Then run:

~~~bash
bunx @every-env/spiral-cli setup --pairing-code YOUR_PAIRING_CODE
~~~

If Spiral gives you an API key instead of a pairing code, run:

~~~bash
bunx @every-env/spiral-cli auth login --token YOUR_SPIRAL_API_KEY
~~~

Verify authentication:

~~~bash
bunx @every-env/spiral-cli auth status
~~~

### 4. Create the MacKenzie style

In the Spiral web app:

1. Go to your chosen workspace.
2. Create a new writing style.
3. Name it MacKenzie.
4. Upload docs/mackenzie-voice-spiral-training-pack.pdf as the source/training document.
5. Let Spiral analyze the PDF and save the style.

List workspaces and styles from the CLI:

~~~bash
bunx @every-env/spiral-cli workspaces --json
bunx @every-env/spiral-cli styles --json
~~~

Copy the workspace ID and style ID:

~~~bash
export SPIRAL_WORKSPACE_ID="your-workspace-id"
export SPIRAL_MACKENZIE_STYLE_ID="your-mackenzie-style-id"
~~~

### 5. Draft directly with Spiral

Use write when Spiral should create the draft from a prompt:

~~~bash
bunx @every-env/spiral-cli write \
  "Write an email from MacKenzie Price to current Alpha families about summer access to TimeBack. Use bracketed placeholders for missing dates, links, and support contacts. No em dashes." \
  --instant \
  --style "$SPIRAL_MACKENZIE_STYLE_ID" \
  --workspace "$SPIRAL_WORKSPACE_ID"
~~~

Use personalize when another AI has already written a draft and Spiral should rewrite it in the trained style:

~~~bash
bunx @every-env/spiral-cli personalize "$(cat draft-email.md)" \
  --style "$SPIRAL_MACKENZIE_STYLE_ID" \
  --workspace "$SPIRAL_WORKSPACE_ID"
~~~

Use humanize when a draft sounds generic or AI-written:

~~~bash
bunx @every-env/spiral-cli humanize "$(cat draft-email.md)" \
  --style "$SPIRAL_MACKENZIE_STYLE_ID" \
  --workspace "$SPIRAL_WORKSPACE_ID"
~~~

Add --json when another agent or script needs parseable output:

~~~bash
bunx @every-env/spiral-cli write \
  "Draft a concise parent email from MacKenzie Price about [topic]. Required facts: [facts]. CTA: [cta]." \
  --instant \
  --json \
  --style "$SPIRAL_MACKENZIE_STYLE_ID" \
  --workspace "$SPIRAL_WORKSPACE_ID"
~~~

## Claude Setup

Claude works well for source-aware drafting, review, and revision. For best output, use Claude to create the baseline draft, then pass that draft through Spiral CLI.

Clone the repo:

~~~bash
git clone git@github.com:PSkinnerTech/mackenzie-voice-tool.git
cd mackenzie-voice-tool
~~~

Install the skill into Claude Code:

~~~bash
mkdir -p ~/.claude/skills
ln -s "$PWD/skills/mackenzie-voice" ~/.claude/skills/mackenzie-voice
~~~

Open Claude Code from the repo root and test the skill:

~~~text
Use the mackenzie-voice skill. Confirm you can see the voice guide, grading schema, and corpus manifest. Do not draft anything yet.
~~~

Ask Claude for a baseline draft:

~~~text
Use the mackenzie-voice skill. Draft an email from MacKenzie Price. Audience: [audience]. Goal: [goal]. Required facts: [facts]. CTA: [cta]. Use bracketed placeholders for missing details. Do not invent anecdotes. No em dashes.
~~~

Save Claude's output to draft-email.md, then run it through Spiral:

~~~bash
bunx @every-env/spiral-cli personalize "$(cat draft-email.md)" \
  --json \
  --style "$SPIRAL_MACKENZIE_STYLE_ID" \
  --workspace "$SPIRAL_WORKSPACE_ID"
~~~

## Perplexity Setup

Perplexity works well for project-space drafting when you want a hosted workspace with uploaded context. For best output, use Perplexity to create a baseline draft, then pass that draft through Spiral CLI.

Create a compact upload file:

~~~bash
python3 mackenzie-price-style/tools/mackenzie_voice_context.py > mackenzie-voice-context-pack.md
~~~

Create a Perplexity Space:

1. Open Perplexity.
2. Go to Spaces.
3. Create a Space named MacKenzie Voice Tool.
4. Upload mackenzie-voice-context-pack.md.
5. Upload docs/mackenzie-voice-spiral-training-pack.pdf if your plan allows it.
6. Add the custom instructions below.

~~~text
You are the MacKenzie Voice Tool. Target voice is MacKenzie Price only. Never draft in Rachel Goodlad's voice or any other Alpha team member's voice.

Use the uploaded MacKenzie corpus, voice guide, grading schema, and training PDF as source context. Outputs are drafts for human review.

Hard rules:
- No em dashes.
- No generic marketing or CRM language.
- Do not invent personal anecdotes.
- Use bracketed placeholders for missing dates, links, locations, people, policies, or CTAs.
- Do not paraphrase canonical Alpha commitments if exact wording is available.
- Prefer concrete parent and kid details over abstract launch language.

Before returning a draft, revise once against those rules.
~~~

Ask Perplexity for a baseline draft:

~~~text
Draft in MacKenzie Price's voice only. Artifact: [email / short blog / social / other]. Audience: [audience]. Goal: [goal]. Required facts: [facts]. CTA: [cta]. Length: [length]. Return the draft and a short self-check.
~~~

Save the draft to draft-email.md, then run it through Spiral:

~~~bash
bunx @every-env/spiral-cli personalize "$(cat draft-email.md)" \
  --json \
  --style "$SPIRAL_MACKENZIE_STYLE_ID" \
  --workspace "$SPIRAL_WORKSPACE_ID"
~~~

## Using Other AI Tools

For ChatGPT, Gemini, Cursor, or any tool without native skill loading:

1. Give the AI the task facts: audience, artifact type, goal, required facts, CTA, length, and anything to avoid.
2. Ask for a plain baseline draft, not a final send.
3. Save the draft locally as draft-email.md.
4. Run the draft through Spiral using the trained MacKenzie style.
5. Review the final output before use.

Baseline prompt:

~~~text
Draft a parent-facing communication. Audience: [audience]. Goal: [goal]. Required facts: [facts]. CTA: [cta]. Use bracketed placeholders for missing operational details. Keep it concise. Do not invent anecdotes. No em dashes.
~~~

Spiral pass:

~~~bash
bunx @every-env/spiral-cli personalize "$(cat draft-email.md)" \
  --json \
  --style "$SPIRAL_MACKENZIE_STYLE_ID" \
  --workspace "$SPIRAL_WORKSPACE_ID"
~~~

## Voice Rules

Target voice is MacKenzie Price only.

Hard rules:

- No em dashes.
- Do not draft in Rachel Goodlad's voice.
- Do not invent personal anecdotes.
- Do not invent dates, links, locations, people, pricing, policies, or commitments.
- Use bracketed placeholders for missing operational facts.
- Avoid generic AI or marketing phrases.
- Treat MacKenzie's direct edits as the highest-signal source.

Canonical Alpha commitments, when used, should keep this wording:

- Kids will love school.
- Kids will learn twice as much in only two hours a day.
- Kids will build life skills.

## Final Review Checklist

- Target voice is MacKenzie only.
- No em dashes.
- No invented anecdotes or operational facts.
- Missing details are bracketed placeholders.
- The draft is concrete, parent-centered, and useful.
- The CTA is clear and low-pressure.
- Canonical Alpha commitments are exact when used.

## Repository Hygiene

Large MP3 source audio files are intentionally excluded from git. The repo stores transcripts, processed corpus, source metadata, examples, docs, and tooling.
